To run the email verifier, use the email_verifier.py script from your terminal:
python email_verifier.py --input_file sample_leads.csv --email_column "Email
Address"
â— Replace sample_leads.csv with the path to your input CSV

â— Replace "Email Address" with the exact name of the column in your CSV that

You can also specify a custom output filename using the --output_file argument:

ğŸ“Š Output Explained
The script will generate a new CSV file (e.g., sample_leads_validated.csv) which
includes all original columns from your input file, plus a new column named
Email_Validation_Status.
Possible values in the Email_Validation_Status column:
â— Valid: The email address passed all syntax, domain, and MX record checks.
â— Invalid Syntax: The email address does not conform to a standard email format.
â— Domain Not Found: The domain part of the email address (e.g., example.com in
user@example.com) does not exist.
â— Domain No A/AAAA Record: The domain exists but has no A or AAAA DNS
records, indicating it might not be a valid host for email.
â— No MX Records: The domain exists but is not configured to receive emails (no Mail
Exchange records found).
â— No Nameservers: No nameservers could be found for the domain.
â— DNS Timeout: The DNS query for the domain timed out.
â— Empty Email: The email field was empty.
â— Invalid Type: The value in the email column was not a string.
â— Error: [message]: An unexpected error occurred during validation.

This tool was developed within a 5-hour time limit, focusing on core, impactful
validation. Therefore, it has certain limitations and areas for future improvement:
â— No Deep SMTP Verification: Does not perform a full SMTP handshake to confirm

â— No Disposable Email Detection: Does not identify temporary or disposable

â— No Catch-All Detection: Does not detect domains configured to accept all
emails (catch-all servers), which can still result in bounces if the specific address
doesn't exist.
â— No Greylisting/Spam Trap Detection: More advanced checks are outside the
scope of basic DNS/MX lookups.
â— Command-Line Interface Only: No graphical user interface.
Potential Future Enhancements:
